前段时间刷到了tsoding和magicalbat的c语言机器学习库，顿时心痒难耐，决定实现一个自己的版本

出发点#

初中时学到了pytorch的autograd，觉得这也太酷了！结果好不容易成为闲得出屁的大学生，直到前段时间才想起自己的梦想。一不做二不休，马上开干

规划#

我没参考tsoding和magicalbat的实现，一个声音告诉我你要是提前偷看答案就输了所以我尝试独立构思。
首先想要实现的是类似pytorch的：

x = torch.relu(self.fc1(x)) # 嵌套函数
x = self.fc2(x) # 复用变量

这意味着矩阵类型必须是一个（逻辑上的）引用，且矩阵数据需要在外部被持有（用于自动微分）
所以矩阵类型mat被定义为mat_s*，而mat_s*：

typedef struct mat_expr {
    mat_op op;
    mat left, right;
} mat_expr;

typedef struct mat_s {
    int rows, cols;
    float *data;
    mat grad;
    mat_expr expr;
    bool use_autograd;
} mat_s;

运算结果矩阵持有参与运算的矩阵，直接由矩阵构成计算图；grad也同为mat类型，为二阶微分做准备
backward就是一个大switch调用当前mat的微分函数，再递归到下一个mat；

遇到的问题#

后向全部做好后，急忙写了一个最简单的异或网络：

mat w1 = mat_rand(2, 2, true);
mat b1 = new_mat(1, 2, true);
mat w2 = mat_rand(2, 1, true);
mat b2 = new_mat(1, 1, true);
mat x = mat_create(1, 2, (float[]){1, 1}, false);
mat y = mat_create(1, 1, (float[]){0}, false);
mat z1 = mat_add(mat_mul(x, w1), b1);
mat a1 = mat_sigmoid(z1);
mat z2 = mat_add(mat_mul(a1, w2), b2);
mat a2 = mat_sigmoid(z2);
mat loss = mat_pow(mat_sub(a2, y), 2);

这只是一次前向过程，那多次前向呢？
其实可以做成动态图，每个循环结束后释放资源，下一次重新构造，优点是可以动态构造复杂网络；然而清除矩阵的实现不太优雅，矩阵之间按照计算图互相持有，必须得从loss往下遍历，还得区分权重/输入矩阵与中间矩阵；所以最后的解决方案是静态图，临时添加了一个全局mat数组，所有运算函数会自动把结果push进去，最后用forward沿着数组执行计算（返回结果矩阵的mat_op自动push，结果矩阵作为参数的mat_set_op同为mat_op内部实现，不push）；
第二个问题是运算的细粒度与随之而来的优化问题：softmax+交叉熵可以被拆成log_softmax+NLLLoss，这种拆法带来了更好的数值稳定性与更好的效率，且二者合并微分形式比拆分的简单得多；将此二者合并又会带来问题：中间矩阵超过了2个，而在一开始的设计中我们只准备了左右值的空位；这里我不得不做了一个hack，把多出来的矩阵存到左值的grad里并设use_autograd为false，避免被zero_grads清除
最后一个问题是在构建mnist手写数字识别网络遇到的；使用普通的[0,1]均匀分布rand的权重mat会导致梯度爆炸，为了解决这个问题学到了xavier初始化，其想法是控制输出方差接近于其输入方差，即：

Var(w)=2nin+nout Var(w) = \frac {2} {n_{in} + n_{out}}

对于均匀分布：

Var(w)=(b−a)212 Var(w) = \frac {(b - a)^2} {12}

我们的分布最后为U(−r,r),r=6nin+noutU(-r, r), r = \sqrt {\frac {6} {n_{in} + n_{out}}}
这个方法也可以缓解梯度消失问题

最后网络完整代码：

mnist_nn 

int mnist_nn() {
    const char *img_path = "train-images.idx3-ubyte";
    const char *lbl_path = "train-labels.idx1-ubyte";

    MnistImages imgs = load_images(img_path);
    MnistLabels lbls = load_labels(lbl_path);

    if (!imgs.data || !lbls.data) {
        fprintf(stderr, "Failed to load dataset.\n");
        return 1;
    }

    printf("Loaded %d images (%dx%d), %d labels.\n\n",
           imgs.num_images, imgs.rows, imgs.cols, lbls.num_labels);
    ;

    srand(time(NULL));
    mat x = mat_rand(784, 1, false);
    mat y = mat_create(1, 1, (float[]){1}, false);

    float bound0 = sqrtf(6.0f / (784 + 16));
    float bound1 = sqrtf(6.0f / (16 + 16));
    float bound2 = sqrtf(6.0f / (16 + 10));

    mat w0 = mat_rand_range(16, 784, -bound0, bound0, true);
    mat w1 = mat_rand_range(16, 16, -bound1, bound1, true);
    mat w2 = mat_rand_range(10, 16, -bound2, bound2, true);

    mat b0 = new_mat(16, 1, true);
    mat b1 = new_mat(16, 1, true);
    mat b2 = new_mat(10, 1, true);

    mat z0 = mat_add(mat_mul(w0, x), b0);
    mat a0 = mat_relu(z0);

    mat z1 = mat_add(mat_mul(w1, a0), b1);
    z1 = mat_relu(z1);
    mat a1 = mat_add(a0, z1);

    mat a2 = mat_add(mat_mul(w2, a1), b2);
    mat loss = mat_softmaxloss(a2, y);
    backward(loss);
    float alpha = 0.01;
    for (int i = 0; i <= 50000; i++) {
        mat_set_image(x, imgs.data + i * 784);
        y->data[0] = lbls.data[i];
        forward(); 
        zero_grads();
        if (i % 10000 == 0) {
            printf("Iter %d: loss=%.4f\n", i, mat_at(loss, 0, 0));
        }
        backward(loss);
        grad_step(w0, alpha);
        grad_step(b0, alpha);
        grad_step(w1, alpha);
        grad_step(b1, alpha);
        grad_step(w2, alpha);
        grad_step(b2, alpha);
    }
    int count = 0;
    for (int i = 50000; i < 60000; i++) {
        mat_set_image(x, imgs.data + i * 784);
        y->data[0] = lbls.data[i];
        forward();
        // printf("Predicted: %d, Actual: %d\n", mat_label(a2), lbls.data[i]);
        if (mat_label(a2) == lbls.data[i]) {
            count++;
        }
    }
    printf("Accuracy: %.2f%%\n", (float)count / 10000 * 100);
    return 0;
}

目前很多操作都还没有封装，之后会继续优化；

改进#

之前遇到的最大问题就是各类矩阵的生命周期问题：参考tsoding和magicalbat的做法，arena是比较好的解决方案；通过arena不但能便捷释放各类矩阵，还便于对各类矩阵进行统一操作（如对所有权重grad_step）目前的想法是定义一个全局的arena指针，通过arena_start/end来划分各个区域；然后考虑是否要将计算图与矩阵分离；
对未来的考虑：目前只实现了全连接和各种运算，如果要实现卷积，就需要考虑张量，因为卷积会输出高一维
